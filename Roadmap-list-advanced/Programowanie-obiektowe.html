Programowanie Obiektowe

Programowanie obiektowe w języku C# polega na tworzeniu klas i obiektów, które reprezentują rzeczywiste modele w aplikacji. Klasy zawierają pola (atrybuty) i metody, które opisują zachowanie obiektów. Obiekty są tworzone na podstawie klas i mogą komunikować się ze sobą poprzez wywoływanie metod. Programowanie obiektowe w C# również obejmuje dziedziczenie, polimorfizm i enkapsulację, co umożliwia tworzenie złożonych i elastycznych aplikacji.

Klasa jest podstawowym elementem programowania obiektowego w języku C#. Jest to szablon, z którego tworzy się obiekty. Klasa zawiera pola (atrybuty), które opisują stan obiektu, oraz metody, które opisują jego zachowanie. 

								
class Car
{
		public string Make;
		public string Model;
		public int Year;

		public void StartEngine()
		{
				Console.WriteLine("Engine started.");
		}
}

Obiekt jest konkretną instancją klasy. Tworzy się go poprzez użycie słowa kluczowego new i nazwę klasy. 

								
Car myCar = new Car();

Dziedziczenie pozwala na tworzenie nowych klas na podstawie już istniejących. Dziedzicząca klasa automatycznie dziedziczy pola i metody klasy bazowej. 

								
class SportsCar : Car
{
		public void StartTurbo()
		{
				Console.WriteLine("Turbo started.");
		}
}

Polimorfizm pozwala na używanie obiektów różnych klas jako obiektów klasy bazowej. 

								
Car myCar = new SportsCar();

myCar.StartEngine();

Enkapsulacja polega na ukrywaniu szczegółów implementacji klasy przed innymi obiektami. Dzięki temu można zmienić implementację klasy bez wpływu na inne obiekty. 

								
class Car
{
		private int _speed;

		public int Speed
		{
				get { return _speed; }
				set { _speed = value; }
		}
}

Interfejsy to kolejny ważny element programowania obiektowego w języku C#. Interfejs jest to zestaw metod, które muszą być zaimplementowane przez klasę, która go implementuje. Interfejsy pozwalają na określenie zachowania, jakie powinien mieć obiekt, bez określania jego implementacji. 

								
interface IDrivable
{
		void Drive();
}

class Car : IDrivable
{
		public void Drive()
		{
				Console.WriteLine("Driving...");
		}
}

abstrakcyjna klasa która jest klasą, która nie może być instancjonowana, lecz może posiadać metody abstrakcyjne, które muszą być zaimplementowane przez klasy pochodne. 

								
abstract class Shape
{
		public abstract double GetArea();
}

class Square : Shape
{
		private int _side;

		public Square(int side)
		{
				_side = side;
		}

		public override double GetArea()
		{
				return _side * _side;
		}
}

Eventy pozwalają na zdefiniowanie zdarzenia, które może wystąpić w aplikacji, i reagowanie na nie. Zdarzenie jest zdefiniowane jako delegat, który określa typ zwracany i parametry, które ma przyjmować metoda obsługi zdarzenia. 

								
class Button
{
		public event EventHandler Click;

		public void OnClick()
		{
				Click?.Invoke(this, EventArgs.Empty);
		}
}

Generyki pozwalają na tworzenie klas, metod lub interfejsów, które przyjmują jako argument typ generyczny. Dzięki temu klasy lub metody mogą pracować z różnymi typami danych, bez konieczności tworzenia oddzielnej implementacji dla każdego z nich. 

								
class List<T>
{
		private T[] _items;

		public void Add(T item)
		{
				// Add item to list
		}

		public T this[int index]
		{
				get { return _items[index]; }
		}
}

Wyjątki (Exceptions) to specjalne obiekty, które służą do obsługi błędów i wyjątków w aplikacji. Służą do informowania o błędach, które wystąpiły podczas działania programu. 

								
try
{
		// Code that might throw an exception
}

catch (Exception ex)
{
		// Code to handle the exception
}

Kolekcje (Collections) to specjalne obiekty, które pozwalają na przechowywanie i zarządzanie danymi w aplikacji. C# posiada wiele gotowych kolekcji, takich jak List, Dictionary, HashSet itp. 

								
List<string> names = new List<string>();

names.Add("bartek");
names.Add("Jan");

Asynchroniczność (Asynchrony) pozwala na wykonywanie operacji w tle, bez blokowania głównego wątku aplikacji. C# posiada składnię asynchronicznej metody, która umożliwia łatwe tworzenie asynchronicznych operacji. 

								
async Task DoWorkAsync()
{
		// Asynchronous code
}

LINQ (Language Integrated Query) to składnia zapytań językowych, która pozwala na łatwe tworzenie zapytań do kolekcji danych. 

								
List<string> names = new List<string>() { "bartek", "Jan", "Bobert" };
var filteredNames = from name in names

										where name.StartsWith("J")

										select name;
